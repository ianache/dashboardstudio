<objective>
Update global CSS tokens and base component classes in `main.css` to enforce a minimalist, professional design with subtle rounding, subdued shadows, and outlined interactive elements.
</objective>

<context>
Phase 49 aims to achieve an airy, minimalist design similar to the Stitch/Google aesthetic.
Deep shadows and heavy solid backgrounds should be replaced with `backdrop-filter`, borders over shadows, and 8px border radii.
@dashboard-app/src/assets/main.css
</context>

<tasks>

<task type="auto">
  <name>Task 1: Subdued Global Shadows, Borders & Layout Tokens</name>
  <files>dashboard-app/src/assets/main.css</files>
  <action>
    Update CSS Custom Properties in `:root` and `[data-theme="light"]` (and dark theme if applicable) to reflect user decisions:
    - Ensure `--radius-md` (or standard radius variables) are set to `0.5rem` (8px).
    - Update `--shadow` and `--shadow-md` to use soft, diffused values (e.g., `0 2px 4px rgba(15, 23, 42, 0.04)`).
    - Ensure a subtle border token (e.g., `--outline`) is defined and used for card/panel separation instead of heavy shadows.
    - Spacing/Density: Increase default padding tokens (e.g. `--spacing-md`, `--spacing-lg`) to create an 'Airy / Relaxed' density.
    - Typography: Define/update heading variables for 'Subtle Hierarchy' and 'Light & Airy' weights (e.g. `--font-weight-heading: 500;`, `--font-weight-body: 300;`).
    - Colors: Update primary and accent colors to 'Soft / Muted Accent' tones (e.g. slate blue or sage green instead of harsh vivid colors).
  </action>
  <verify>
    <automated>grep -q "0.5rem\|8px" dashboard-app/src/assets/main.css || (echo "Missing radius update" && exit 1)</automated>
  </verify>
  <done>Tokens for radius, shadows, typography, and spacing are updated to minimalist specs.</done>
</task>

<task type="auto">
  <name>Task 2: Outlined Inputs & Buttons</name>
  <files>dashboard-app/src/assets/main.css</files>
  <action>
    Refactor base button classes (`.btn-primary`, `.btn-secondary`, etc.) and input classes to an "outlined" aesthetic.
    - Set background to transparent or surface color.
    - Add subtle borders using `var(--primary)` for primary buttons and `var(--outline)` for secondary.
    - Use `color-mix()` for hover states (e.g., `background: color-mix(in srgb, var(--primary) 10%, transparent);`).
    - Update text color to avoid harsh black; use `var(--on-surface-variant)` where appropriate.
  </action>
  <verify>
    <automated>grep -q "color-mix" dashboard-app/src/assets/main.css || (echo "Missing color-mix hover state" && exit 1)</automated>

</task>
