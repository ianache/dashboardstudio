<objective>
Migrate iconography from Material Symbols to Lucide (line-art style) by converting `MIcon.vue` into a dynamic adapter. This avoids a massive codebase refactor while achieving the minimalist aesthetic.
</objective>

<context>
Phase 49 requires Light Line-Art Icons. `lucide-vue-next` is the chosen standard.
We will use an adapter pattern in `MIcon.vue` to map existing Material icon strings (e.g., 'grid_view') to Lucide components (e.g., `LayoutGrid`).
</context>

  <name>Task 2: Refactor MIcon Adapter</name>
  <files>dashboard-app/src/components/common/MIcon.vue</files>
  <action>
    Rewrite `MIcon.vue` to act as a wrapper for Lucide icons.
    - Import `computed` from `vue` and `iconMap` from `./IconMap.js`.
    - Import a fallback icon (e.g., `HelpCircle`) from `lucide-vue-next`.
    - Dynamically resolve the icon component based on the `icon` prop, using the fallback if not found.
    - Translate the legacy `weight` prop into a `stroke-width` (e.g., `weight > 400 ? 2 : 1.5`).
    - Set the default color of the icon to `var(--on-surface-variant)` unless explicitly overridden by props.
    - Render `<component :is="resolvedIcon" :size="size" :stroke-width="strokeWidth" :color="color || 'var(--on-surface-variant)'" ... />`.
    - Remove old material-symbols font classes or link tags if they were hardcoded in this component.
