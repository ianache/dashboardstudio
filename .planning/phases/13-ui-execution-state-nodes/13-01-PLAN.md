# Plan 13-01: UI Execution State & Node Visuals

Este plan implementa el manejo del estado de ejecución en el frontend y actualiza la visualización de los nodos en el canvas.

## Goals
1. Mantener un mapeo reactivo de `node_id` -> `status` en el Flow Editor.
2. Aplicar estilos dinámicos a los nodos según su estado (borde verde grueso para `running`).
3. Mostrar badges de éxito (check) o error (cross) en los nodos completados.

## Tasks

### 1. State Management
- En `FlowEditorCanvas.vue`, añadir un objeto reactivo `nodeExecStatus = ref({})`.
- En la función `runFlow`, resetear `nodeExecStatus.value = {}` al inicio.
- En el listener `ws.onmessage`, si el evento es de tipo `node_status`, actualizar `nodeExecStatus.value[data.node_id] = data.status`.

### 2. Node Visual Highlighting
- Modificar el template del nodo en el canvas para aplicar clases dinámicas:
    - `:class="{ 'fec-node--executing': nodeExecStatus[node.id] === 'running', 'fec-node--success': nodeExecStatus[node.id] === 'success', 'fec-node--error': nodeExecStatus[node.id] === 'error' }"`.
- Añadir CSS para estas clases:
    - `.fec-node--executing`: Borde verde de 3px, sombra suave verde.
    - `.fec-node--success`: Borde verde normal.
    - `.fec-node--error`: Borde rojo normal.

### 3. Node Badges
- Añadir un elemento badge en la esquina del nodo.
- El badge debe mostrarse solo si el estado es `success` o `error`.
- Usar iconos de Material Symbols: `check_circle` para éxito, `cancel` para error.
- Añadir CSS para posicionar los badges y darles color.

## Verification
1. Ejecutar un flujo desde la UI.
2. Verificar que el nodo fuente se resalta en verde mientras corre.
3. Verificar que al terminar el primer nodo, aparece el badge de check y el siguiente nodo se resalta.
4. Verificar que el estado se limpia correctamente al iniciar una nueva ejecución.
