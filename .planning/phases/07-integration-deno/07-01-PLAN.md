# Plan 07-01: Deno Infrastructure & Runner

Este plan establece la base técnica para la ejecución de scripts en los flujos de integración.

## Goals
1. Crear el script `runner.ts` que actuará como orquestador dentro de Deno.
2. Implementar `DenoService` en el backend para gestionar subprocesos de Deno.
3. Validar la comunicación entre FastAPI y Deno con una ejecución de prueba.

## Tasks

### 1. Runner Script (Deno)
- Crear `backend/app/runtime/runner.ts`.
- El script debe:
    - Leer un JSON desde `stdin`.
    - Soporta un modo "test" para validar que el runtime está activo.
    - Implementar una función `executeNode` básica que use `new Function` para ejecutar código JS.
    - Capturar y formatear la salida de `console.log`.

### 2. Deno Service (FastAPI)
- Crear `backend/app/services/deno_service.py`.
- Implementar la clase `DenoService`:
    - `check_runtime()`: Verifica que `deno --version` responda.
    - `run_flow(flow_data: dict)`: Ejecuta el subproceso con los flags de seguridad:
        - `--no-remote`
        - `--allow-net` (opcionalmente deshabilitado por ahora)
        - `--v8-flags=--max-old-space-size=256`
    - Manejar timeouts (ej: 10 segundos para este MVP).

### 3. API Integration
- Añadir un endpoint temporal en `backend/app/api/endpoints/integration_flows.py`:
    - `POST /integration-flows/check-runtime`: Devuelve la versión de Deno y una prueba de ejecución de script.

## Verification
1. Ejecutar el endpoint de check y verificar que el "Hello from Deno" aparezca en la respuesta.
2. Comprobar que el timeout funciona si se envía un script con un bucle infinito.
3. Verificar que los límites de memoria son respetados.
