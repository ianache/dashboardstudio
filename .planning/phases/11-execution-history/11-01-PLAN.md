# Plan 11-01: Execution History & Security Hardening

Este plan añade persistencia al historial de ejecuciones y refuerza la seguridad del motor.

## Goals
1. Registrar cada ejecución de flujo en la base de datos con sus logs.
2. Permitir visualizar el historial de ejecuciones en la UI.
3. Asegurar que los timeouts y límites de recursos son infranqueables.

## Tasks

### 1. Backend: Model & Schema
- Actualizar `backend/app/models/models.py`:
    - Crear clase `IntegrationFlowExecution`.
    - Campos: `id`, `flow_id`, `status`, `logs`, `result_data`, `duration_ms`, `created_at`.
- Actualizar `backend/app/schemas/schemas.py`:
    - Añadir esquemas para `IntegrationFlowExecution`.

### 2. Backend: Execution Persistence
- Actualizar el endpoint WebSocket y el POST run en `integration_flows.py`.
- Guardar un registro en `IntegrationFlowExecution` al finalizar cada ejecución.
- Almacenar los logs completos capturados del stream.

### 3. Backend: Execution History API
- Añadir endpoint `GET /integration-flows/{flow_id}/executions`.
- Añadir endpoint `GET /integration-flows/executions/{exec_id}/logs`.

### 4. Frontend: History Tab
- Añadir una pestaña "Historial" en el panel derecho de `FlowEditorCanvas.vue`.
- Listar las últimas ejecuciones con su estado y fecha.
- Permitir hacer clic en una ejecución para cargar sus logs en la consola.

### 5. Hardening: Timeouts
- Asegurar que el timeout de 30 segundos (o configurable) se aplica rigurosamente en `DenoService`.
- Verificar que el proceso Deno es "matado" (`SIGKILL`) si no termina.

## Verification
1. Ejecutar un flujo y verificar que aparece una nueva fila en `integration_flow_executions`.
2. Ver el historial en la UI y cargar los logs de una ejecución pasada.
3. Ejecutar un script con `while(true){}` y verificar que el sistema lo detiene y registra el error de timeout.
