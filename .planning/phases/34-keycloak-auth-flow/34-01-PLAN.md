# Plan: 34-01-PLAN - ESM Migration & OIDC Client Setup

## Goal
Convert the BFF to an ES Module (ESM) and set up the `openid-client` v6 infrastructure. This is a prerequisite for using `openid-client` v6, which is ESM-only.

## Proposed Changes

### 1. `bff/package.json`
- Add `"type": "module"`.
- Add `openid-client` to dependencies.

### 2. `bff/src/config.js`
- Convert to ESM (use `export default`).

### 3. `bff/src/session.js`
- Convert to ESM (use `import`, `export`).

### 4. `bff/src/routes/health.js`
- Convert to ESM.

### 5. `bff/src/oidc.js` (NEW)
- Implement OIDC discovery using `openid-client`.
- Export the `config` object (after discovery).

### 6. `bff/src/index.js`
- Convert to ESM.
- Initialize OIDC client before starting the server.

## Step-by-Step Execution

### Step 1: Update `package.json`
```bash
cd bff
npm pkg set type=module
npm install openid-client
```

### Step 2: Convert `config.js`
- Replace `module.exports = ...` with `export default ...`.
- Replace `require()` with `process.env`.

### Step 3: Convert `session.js`
- Replace `require()` with `import`.
- Export `sessionMiddleware` and `redisClient`.

### Step 4: Convert `routes/health.js`
- Replace `require()` with `import`.
- Export the router.

### Step 5: Implement `oidc.js`
- Use `oidc.discovery()` to get the provider configuration.
- Handle potential discovery errors.

### Step 6: Convert `index.js`
- Use `import`.
- Wait for OIDC discovery before calling `app.listen()`.
