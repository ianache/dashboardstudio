# Plan 15-01: Backend - Connection Data Model

Este plan implementa la estructura de datos para la gestión centralizada de conexiones, incluyendo la encriptación recursiva y la validación polimórfica.

## Goals
1. Extender el modelo `DataSource` con `connection_config`.
2. Implementar esquemas Pydantic con Discriminated Unions.
3. Añadir utilidades de encriptación recursiva.

## Tasks

### 1. Encryption Utilities
- Actualizar `backend/app/core/encryption.py` con `process_sensitive_fields(data, action)`.
- Asegurar que soporta recursividad para diccionarios y listas.

### 2. Pydantic Schemas
- Crear `backend/app/schemas/connection_schemas.py`.
- Definir modelos base y específicos (`SmtpConfig`, `DbConfig`, `FtpConfig`, `HttpConfig`, `JwtConfig`).
- Utilizar `Annotated` y `Field(discriminator='type')`.

### 3. Database Model Update
- Modificar `DataSource` en `backend/app/models/models.py`.
- Añadir `connection_config = Column(JSON, default=dict)`.
- Crear migración (Alembic) para aplicar este cambio.

### 4. API Refactoring
- Actualizar CRUD de `DataSources` para usar los nuevos esquemas.
- Integrar `process_sensitive_fields` en la capa de persistencia (antes de salvar/después de leer).

## Verification
1. Crear un script que valide la serialización/deserialización de cada tipo de conexión.
2. Verificar que los campos sensibles (ej: `password`) estén encriptados en la base de datos (raw query).
3. Asegurar que los endpoints devuelven la configuración desencriptada correctamente.
