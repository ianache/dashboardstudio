# Plan 16-01: Backend - Connection Testing Logic

Implementar el patrón Strategy para validar conexiones de forma asíncrona.

## Goals
1. Crear una interfaz/clase base `BaseConnectionStrategy`.
2. Implementar estrategias para: `smtp`, `database`, `ftp`, `http`.
3. Exponer endpoint `/api/v1/data-sources/{id}/test`.

## Tasks

### 1. Strategy Pattern
- Crear `backend/app/services/connection_testing.py`.
- Definir `BaseConnectionStrategy` con método `test()`.
- Implementar `SmtpStrategy`, `DbStrategy`, `FtpStrategy`, `HttpStrategy`.
- Usar bibliotecas adecuadas: `smtplib`, `asyncpg` (o `sqlalchemy`), `ftplib`.

### 2. API Endpoint
- Crear `/api/v1/data-sources/{id}/test`.
- El endpoint debe:
    - Recuperar el DataSource.
    - Desencriptar el JSON `connection_config` usando `process_sensitive_fields(..., "decrypt")`.
    - Seleccionar la estrategia basada en `config['type']`.
    - Ejecutar `strategy.test()`.

### 3. Verification
- Crear script de test que valide un DataSource existente.
- Verificar manejo de errores (timeouts, auth fallida).
