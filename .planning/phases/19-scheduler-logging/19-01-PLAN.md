# Plan 19-01: Backend - Scheduler & Logging Models

Implementar modelos de persistencia para historial de ejecución e integrar el scheduler en background.

## Goals
1. Definir modelos `ExecutionHistory` y `NodeExecutionLogs`.
2. Integrar `APScheduler` en FastAPI.
3. Actualizar `IntegrationFlow` para incluir campos de cron y logs.

## Tasks

### 1. Database Models
- Añadir `IntegrationFlow.cron_expression` (String).
- Añadir `IntegrationFlow.log_level` (String, default='summary').
- Crear `ExecutionHistory` (id, flow_id, status, start_time, end_time, duration).
- Crear `NodeExecutionLogs` (id, execution_id, node_id, status, start_time, end_time, duration, input_data, output_data).

### 2. Scheduler Integration
- Crear `backend/app/services/scheduler.py` con instancia de `BackgroundScheduler`.
- Añadir lógica para cargar tareas programadas desde la BD al iniciar la aplicación.
- Configurar evento de inicio en `app/main.py`.

### 3. Verification
- Ejecutar migración de DB.
- Verificar que el scheduler inicia correctamente.
- Probar un flujo con cronjob.
