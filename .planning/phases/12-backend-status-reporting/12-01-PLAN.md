# Plan 12-01: Backend Status Reporting

Este plan implementa la emisión de estados de nodo desde Deno y su procesamiento en FastAPI para habilitar la visualización en tiempo real.

## Goals
1. Actualizar `runner.ts` para emitir tokens de estado por nodo.
2. Actualizar `deno_service.py` para detectar estos tokens y transformarlos en eventos de WebSocket.
3. Validar que los estados `running`, `success` y `error` fluyen correctamente.

## Tasks

### 1. Runner: Node Status Emission
- Actualizar `backend/app/runtime/runner.ts`.
- Añadir una función `emitStatus(nodeId: string, status: 'running' | 'success' | 'error')`.
- La función debe imprimir `NODE_STATUS:${nodeId}:${status}` en `console.log`.
- Integrar `emitStatus` en el bucle principal de ejecución:
    - Emitir `running` antes de ejecutar cada nodo.
    - Emitir `success` después de una ejecución exitosa.
    - Emitir `error` si el nodo falla.

### 2. Deno Service: Status Token Parsing
- Actualizar `backend/app/services/deno_service.py`.
- En el método `run_flow_stream`, añadir lógica para detectar líneas que comiencen con `NODE_STATUS:`.
- Si se detecta el token, no emitirlo como `info` log normal, sino como un evento estructurado:
    - `yield {"type": "node_status", "node_id": "...", "status": "..."}`.

## Verification
1. Ejecutar un flujo de prueba usando el script de diagnóstico `verify_streaming.py` (actualizado para esperar eventos `node_status`).
2. Verificar que se reciben eventos en este orden: `running` -> `success` (o `error`) para cada nodo.
3. Asegurar que los logs normales (no status) se siguen recibiendo correctamente.
