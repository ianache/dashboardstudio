## Critical Pitfalls to Address (from PITFALLS.md):

### Pitfall 1: Deadlocks in Concurrent Upserts
- **Prevention:** Always order operations by conflict keys before upsert
- **Implementation:** Sort records by identity_fields before batch execution
- **Retry:** Implement exponential backoff for deadlock errors (SQLSTATE '40P01')

### Pitfall 2: Transaction Scope Violations  
- **Prevention:** Each batch is self-contained with its own transaction
- **Implementation:** Use `async with conn.transaction():` per batch
- **Never span transactions across multiple executor calls**

### Pitfall 3: JSON Serialization Failures
- **Prevention:** Use `allow_nan=False` in Python JSON parsing
- **Dates:** ISO 8601 format only
- **BigInt:** Serialize as strings

### Pitfall 4: Memory Exhaustion
- **Prevention:** Process in configurable batches (default: 1000 rows)
- **Implementation:** Iterate through batches, don't load all into memory

### Pitfall 10: Subprocess Communication (handled in Plan 31-02)
- **Note:** This plan creates the executor that will be called from DenoService

## Architecture Decisions (from ARCHITECTURE.md):

### Per-Batch Transaction Strategy (Recommended)
```python
async with conn.transaction():
    result = await conn.executemany(query, batch)
    # Each batch is atomic, failures isolated
```

### Error Classification
| Error Type | Action | Retry? |
|------------|--------|--------|
| Connection Error | Abort all | Yes (3x) |
| Unique Violation | Skip record | No |
| FK Violation | Skip record | No |
| Type Mismatch | Skip record | No |
| Timeout | Retry batch | Yes (3x) |
| Deadlock | Retry batch | Yes (3x) |

2. **Main execute() method:**
   ```python
   async def execute(
       self,
       config: ODSConfig,
       records: List[Dict[str, Any]],
       connection: asyncpg.Connection
   ) -> ODSResult:
   ```
   - Validate records is non-empty list
   - Calculate total duration
   - Split records into batches based on config.batch_size
   - Process batches sequentially (parallel deferred to v2)
   - Aggregate results from all batches
   - Return ODSResult with complete statistics

3. **_execute_batch() helper:**
   - Takes a single batch, executes appropriate operation
   - Wraps each batch in its own transaction
   - Returns batch results or error info

4. **Append operation (_execute_append):**
   - Generate INSERT INTO statement with parameterized placeholders
   - Use executemany() for batch insert
   - Return rows inserted count
   - SQL pattern: `INSERT INTO "schema"."table" (cols) VALUES ($1, $2, ...)`

5. **Overwrite operation (_execute_overwrite):**
   - TRUNCATE TABLE within transaction BEFORE inserts
   - Then perform append operation
   - SQL: `TRUNCATE TABLE "schema"."table" RESTART IDENTITY`
   - Note: RESTART IDENTITY resets sequences

6. **Upsert operation (_execute_upsert):**
   - Generate INSERT ON CONFLICT statement
   - Conflict target: identity fields (columns specified in config.identity_fields)
   - DO UPDATE SET for all non-identity columns
   - **CRITICAL:** Sort records by identity_fields before processing to prevent deadlocks (Pitfall 1)
   - SQL pattern: 
     ```sql
     INSERT INTO "schema"."table" (cols) VALUES ($1, $2, ...)
     ON CONFLICT (id_field1, id_field2) DO UPDATE SET
       col1 = EXCLUDED.col1,
       col2 = EXCLUDED.col2
     ```

7. **Parameterized query generation:**
   - Never use string interpolation for values
   - Use asyncpg's native parameterization
   - For table/schema names: use proper quoting `"schema"."table"`

8. **Retry logic with exponential backoff:**
   - Retry on: ConnectionError, Timeout, Deadlock
   - Max 3 retries with backoff: 1s, 2s, 4s
   - Use asyncio.sleep between retries
   - Log each retry attempt

9. **Error handling per batch:**
   - Catch asyncpg exceptions
   - Classify error type using SQLSTATE
   - Log detailed error with batch number
   - Store error in ODSResult.errors list
   - Failed batch rolls back, continue with next batch

**Key implementation details from PITFALLS.md:**
- Always use `async with conn.transaction():` for each batch (Pitfall 2)
- Sort records by identity_fields before upsert (Pitfall 1)
- Use executemany() with parameterized queries (SQL injection prevention)
- Set statement_timeout per operation type:
  - append: '5min'
  - overwrite: '30min' 
  - upsert: '10min'

**Security:**
- Never interpolate user data into SQL
- Quote identifiers using asyncpg's built-in methods if available, or proper double-quote escaping
