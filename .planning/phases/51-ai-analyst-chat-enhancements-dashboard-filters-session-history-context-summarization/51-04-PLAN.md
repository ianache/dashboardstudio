<objective>
Pass request.model to _summarize_session() so the context summarization uses the same inference model the user selected for the BI AI Analyst conversation. Currently the function hardcodes "gemini-2.5-flash-lite" regardless of the active model. For Gemini models, use the selected model directly via genai.Client(). For non-Gemini models (DeepSeek, Groq, etc.), fall back to gemini-2.5-flash-lite since they require ADK runner infrastructure not available in the one-shot summarization path.

Purpose: Consistency — if the user chose gemini-2.5-pro-preview for analysis, the summary should also use that model. Avoids silent model-switching without user knowledge.

Output: _summarize_session() signature extended with model parameter. chat() endpoint passes request.model. Log line shows which model performed the summarization.
</objective>
