<critical_pitfalls>
Pitfalls from research to avoid during implementation:

1. APP_NAME mismatch: Define `APP_NAME = "ai-analyst"` as a module-level constant in agent.py. Pass it to both `Runner(app_name=APP_NAME)` AND every `session_service.create_session(app_name=APP_NAME)`. Any mismatch causes "Session not found" errors.

2. Agent name with hyphens: Use `name="bi_analyst"` (underscores), NOT `name="bi-analyst"`. ADK requires valid Python identifiers.

3. StreamingMode not passed: Always construct `run_config = RunConfig(streaming_mode=StreamingMode.SSE)` and pass it to every `runner.run_async()` call. Without it, ADK defaults to non-streaming and the UI sees one large response at the end.

4. Mid-stream errors: Wrap the entire async generator body in try/except. On exception, yield `ServerSentEvent(data=json.dumps({"type": "error", "message": str(e)}))`. Do NOT let the exception propagate (which would drop the SSE connection with an HTTP error).

5. usage_metadata guard: Always check `if event.usage_metadata:` before accessing `event.usage_metadata.prompt_token_count`. The field may be None on some final events.

6. GOOGLE_GENAI_USE_VERTEXAI: ADK reads this from env automatically. As long as .env-ai-analyst sets `GOOGLE_GENAI_USE_VERTEXAI=FALSE`, ADK uses the API key route. Config.py stores it for documentation; the env var itself is what ADK checks.

7. FastAPI SSE import path: `from fastapi.sse import EventSourceResponse, ServerSentEvent` — requires fastapi>=0.135.0. If import fails at runtime (google-adk pins an older fastapi), fall back to `from sse_starlette.sse import EventSourceResponse` and add `sse-starlette` to pyproject.toml.
</critical_pitfalls>
</context>

Google ADK agent singleton.

The Runner and session_service are module-level singletons — safe for concurrent
reuse per ADK design (discussion #3924). Only session_id varies per request.

from google.adk.agents import LlmAgent
from google.adk.runners import Runner
from google.adk.sessions import InMemorySessionService

from app.core.config import get_settings

APP_NAME = "ai-analyst"  # Must match in all session_service calls

settings = get_settings()

session_service = InMemorySessionService()

root_agent = LlmAgent(
    name="bi_analyst",  # underscores required — ADK validates Python identifier
    model=settings.gemini_model,
    instruction=(
        "You are a BI analyst assistant embedded in a dashboard designer tool. "
        "Answer questions about business data clearly and concisely. "
        "When you do not have enough data context to answer precisely, say so honestly."
    ),
)

runner = Runner(
    app_name=APP_NAME,
    agent=root_agent,
    session_service=session_service,
)
```

- `settings.gemini_model` reads GEMINI_MODEL from env — no hardcoded model name.
- Agent name uses underscores (`bi_analyst`) — ADK requires valid Python identifiers.
- Phase 44 extends `root_agent` with tools (CubeJS, skills catalog) by adding `tools=[...]` to this LlmAgent definition. Keep the module boundary clean.

"""
AI Analyst FastAPI application.

Endpoints:
  GET  /health  → {status: ok}
  POST /chat    → SSE stream of {type: token|done|error, ...} events
"""
import json
import uuid

from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

from google.genai import types
from google.adk.agents.run_config import RunConfig, StreamingMode

from app.core.config import get_settings

# Try native FastAPI SSE (>=0.135.0); fall back to sse-starlette if unavailable
try:
    from fastapi.sse import EventSourceResponse, ServerSentEvent
except ImportError:
    from sse_starlette.sse import EventSourceResponse, ServerSentEvent  # type: ignore[no-redef]

from app.agent import APP_NAME, runner, session_service

settings = get_settings()

app = FastAPI(title="AI Analyst Service", version="0.1.0")

app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.cors_origins.split(","),
    allow_methods=["GET", "POST"],
    allow_headers=["*"],
)


class ChatRequest:
    def __init__(self, message: str):
        self.message = message


# Use Pydantic model for FastAPI body parsing
from pydantic import BaseModel

class ChatRequest(BaseModel):
    message: str


@app.get("/health")
async def health():
    return {"status": "ok"}


@app.post("/chat")
async def chat(request: ChatRequest):
    session_id = str(uuid.uuid4())

    # Create a fresh session per request — Phase 43 has no conversation history
    await session_service.create_session(
        app_name=APP_NAME,
        user_id="default",
        session_id=session_id,
    )

    async def stream():
        run_config = RunConfig(streaming_mode=StreamingMode.SSE)
        try:
            async for event in runner.run_async(
                user_id="default",
                session_id=session_id,
                new_message=types.Content(
                    role="user",
                    parts=[types.Part(text=request.message)],
                ),
                run_config=run_config,
            ):
                # Emit token events for partial text chunks
                if event.content and event.content.parts:
                    for part in event.content.parts:
                        if hasattr(part, "text") and part.text:
                            yield ServerSentEvent(
                                data=json.dumps({"type": "token", "text": part.text})
                            )

                # Emit done event on final response with usage metadata
                if event.is_final_response():
                    usage = {"input_tokens": 0, "output_tokens": 0}
                    if event.usage_metadata:
                        usage["input_tokens"] = getattr(
                            event.usage_metadata, "prompt_token_count", 0
                        ) or 0
                        usage["output_tokens"] = getattr(
                            event.usage_metadata, "candidates_token_count", 0
                        ) or 0
                    yield ServerSentEvent(
                        data=json.dumps({"type": "done", "usage": usage})
                    )

        except Exception as e:
            # Inline error — do not disconnect with HTTP 500
            yield ServerSentEvent(
                data=json.dumps({"type": "error", "message": str(e)})
            )

    return EventSourceResponse(stream())
```

Key implementation notes:
- `ChatRequest` defined once as a Pydantic BaseModel (remove the intermediate `__init__` version — it is a drafting artifact, use only the `BaseModel` subclass).
- `getattr(event.usage_metadata, "prompt_token_count", 0)` — guards against field name variation between ADK versions (per open question in research).
- `settings.cors_origins.split(",")` — reads comma-separated origins from env.
- The duplicate `class ChatRequest` in the draft above: keep ONLY the `BaseModel` version. The `__init__` version is a drafting artifact that must not appear in the final file.
