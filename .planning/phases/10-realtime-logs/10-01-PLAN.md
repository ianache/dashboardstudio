# Plan 10-01: Real-time Logs & Monitoring UI

Este plan implementa el streaming de logs por WebSockets y el componente de consola en la UI.

## Goals
1. Transmitir logs de Deno en tiempo real desde el backend.
2. Crear un componente de consola interactiva en el Flow Editor.
3. Permitir al usuario iniciar la ejecución y ver el progreso en vivo.

## Tasks

### 1. Deno Service: Streaming Support
- Actualizar `backend/app/services/deno_service.py`.
- Implementar `run_flow_stream(flow_data: dict, payload: dict)`:
    - Utilizar `asyncio.create_subprocess_exec`.
    - Leer `stdout` y `stderr` línea por línea usando un generador asíncrono.
    - Emitir eventos de log con tipo (info, error) y contenido.

### 2. API: WebSocket Endpoint
- Actualizar `backend/app/api/endpoints/integration_flows.py`.
- Implementar `WS /integration-flows/{flow_id}/logs`.
- El endpoint debe:
    - Aceptar conexión WebSocket.
    - Recibir un mensaje inicial con el `payload`.
    - Llamar a `deno_service.run_flow_stream`.
    - Enviar cada línea de log al cliente en formato JSON.
    - Enviar un mensaje final con el resultado de la ejecución.

### 3. Frontend: ExecutionConsole Component
- Crear `dashboard-app/src/components/editor/ExecutionConsole.vue`.
- El componente debe:
    - Mostrar una terminal oscura.
    - Listar logs con colores (rojo para errores, azul para info).
    - Mostrar estado de ejecución (Running, Success, Error).
    - Botón para limpiar logs.

### 4. Frontend: FlowEditorCanvas Integration
- Integrar `ExecutionConsole` en un nuevo panel inferior colapsable.
- Añadir un botón "Ejecutar" en el panel de herramientas o de propiedades.
- Gestionar la conexión WebSocket y el paso de datos al componente de consola.

## Verification
1. Abrir un flujo, hacer clic en "Ejecutar".
2. Verificar que el panel de consola se abre automáticamente.
3. Observar cómo aparecen los logs de "Starting flow", "Executing Node", etc., en tiempo real.
4. Verificar que al finalizar se muestra el resultado JSON y el estado cambia a "Success".
