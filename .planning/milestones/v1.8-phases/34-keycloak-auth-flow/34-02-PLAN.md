# Plan: 34-02-PLAN - Auth Routes

## Goal
Implement the core authentication routes in the BFF: login, callback, me, and logout. These routes will handle the OIDC flow and session management.

## Proposed Changes

### 1. `bff/src/routes/auth.js` (NEW)
- Implement `GET /auth/login`:
  - Generate PKCE code verifier and challenge.
  - Store verifier in session.
  - Build and redirect to Keycloak authorization URL.
- Implement `GET /auth/callback`:
  - Extract code from URL.
  - Exchange code for tokens using `openid-client` and the verifier from session.
  - Store tokens (access, refresh, id) in session.
  - Redirect to a success page (or frontend).
- Implement `GET /auth/me`:
  - Return user info from tokens stored in session (sub, email, name, roles).
  - Return 401 if no valid session/tokens.
- Implement `GET /auth/logout`:
  - Revoke tokens (optional but good).
  - Destroy BFF session.
  - Redirect to Keycloak end-session endpoint.

### 2. `bff/src/index.js`
- Mount the new `authRouter` at `/bff/auth`.

## Step-by-Step Execution

### Step 1: Create `bff/src/routes/auth.js`
- Use `getOIDCConfig()` from `oidc.js`.
- Use `oidc.buildAuthorizationUrl`, `oidc.authorizationCodeGrant`.

### Step 2: Implement User Info logic
- Extract claims from ID token or Access Token.
- Keycloak roles are usually in `resource_access` or `realm_access`.
