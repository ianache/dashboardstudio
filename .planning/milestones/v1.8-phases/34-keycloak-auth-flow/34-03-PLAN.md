# Plan: 34-03-PLAN - Token Management & Refresh

## Goal
Implement middleware to manage token lifecycle, ensuring they are automatically refreshed before they expire.

## Proposed Changes

### 1. `bff/src/middleware/auth.js` (NEW)
- Implement `requireAuth`:
  - Checks if `req.session.user` exists.
  - Returns 401 if not.
- Implement `tokenRefresh`:
  - Checks if `req.session.tokens` exists.
  - Compares `expires_at` with current time (with a buffer, e.g., 30 seconds).
  - If expired/expiring, calls `oidc.refreshTokenGrant()`.
  - Updates `req.session.tokens` and `req.session.user` (claims might have changed).

### 2. `bff/src/routes/auth.js`
- Apply `tokenRefresh` to `/me`.

## Step-by-Step Execution

### Step 1: Create `bff/src/middleware/auth.js`
- Use `oidc.refreshTokenGrant` from `openid-client`.

### Step 2: Update `/me` route
- Add the middleware to the route definition.
