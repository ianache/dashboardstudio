# Plan: 37-01-PLAN - Refactor Auth, API, and CubeJS

Refactor the frontend state management and API services to use the BFF instead of direct token-based communication.

## Tasks

<task id="37-01-01">
  <name>Refactor Auth Store</name>
  <files>["dashboard-app/src/stores/auth.js"]</files>
  <action>
    - Remove keycloak-js dependency and _keycloak reference.
    - Add `initialized` boolean flag to state (default: false).
    - Implement initialize() action:
      - Call fetch(import.meta.env.VITE_BFF_URL + '/bff/auth/me').
      - On 200: Populate `user` from response.
      - On 401 or failure: Set `user = null` and redirect to `import.meta.env.VITE_BFF_URL + '/bff/auth/login'`.
      - Set `initialized = true` in finally block.
    - Update logout() to redirect to `import.meta.env.VITE_BFF_URL + '/bff/auth/logout'`.
    - Remove token-related getters and actions.
  </action>
  <verify>
    <automated>grep "initialized" dashboard-app/src/stores/auth.js</automated>
  </verify>
  <done>Auth store migrated to BFF-driven model with initialization tracking.</done>
</task>

<task id="37-01-02">
  <name>Refactor API Service</name>
