  <name>Task 1: Create bff/ directory structure with package.json and Dockerfile</name>
  <files>bff/package.json, bff/.dockerignore, bff/Dockerfile</files>
  <action>
Create the `bff/` directory at the repo root (same level as `backend/` and `dashboard-app/`).

**bff/package.json** — name: "dashboard-bff", version: "1.0.0", main: "src/index.js", no TypeScript. Scripts: `"start": "node src/index.js"`. Dependencies: `express` (latest, resolves to 5.x), `express-session`, `connect-pg-simple`, `pg`. DevDependencies: `dotenv` (for local dev only). Do NOT add a `type: "module"` field — use CommonJS (`require`/`module.exports`) throughout for compatibility with connect-pg-simple 10.x.

**bff/.dockerignore** — exclude: `node_modules/`, `.env-bff`, `npm-debug.log`, `*.md`. This prevents local node_modules from being sent to Docker build context and prevents .env-bff from leaking into the image.

**bff/Dockerfile** — Single-stage, `FROM node:20-alpine`. Steps:
1. `WORKDIR /app`
2. `COPY package*.json ./`
3. `RUN npm install --omit=dev` (production deps only)
4. `COPY src/ ./src/`
5. `ENV NODE_ENV=production`
6. `EXPOSE 3001`
7. `HEALTHCHECK --interval=30s --timeout=5s --start-period=30s --retries=3 CMD wget -qO- http://localhost:3001/bff/health || exit 1` (use `wget` — curl is NOT in alpine by default)
8. `CMD ["node", "src/index.js"]`

Do NOT add a multi-stage build — this is a server, not a frontend bundle.
  </action>

<task type="auto">
  <name>Task 2: Create Express 5 app source — config.js, health route, index.js</name>
  <files>bff/src/config.js, bff/src/routes/health.js, bff/src/index.js</files>
  <action>
Create `bff/src/`, `bff/src/routes/` directories and the three source files.

**bff/src/config.js** — Fail-fast env var validation. All values come from `process.env` — zero hardcoded URLs or secrets. Use a `required(key)` helper that throws `new Error(`Missing required env var: ${key}`)` if the value is falsy. Export a `module.exports = Object.freeze({ ... })` with all BFF_* vars parsed. Optional vars (BFF_PORT, BFF_POSTGRES_PORT, BFF_POSTGRES_SCHEMA) use `process.env.VAR || 'default'`. This module is the single source of truth — all other modules `require('./config')` instead of reading process.env directly.

**bff/src/routes/health.js** — Express 5 Router. Single route: `router.get('/health', (req, res) => res.json({ status: 'ok' }))`. Export the router via `module.exports = router`. No auth, no session check — health endpoint must be publicly accessible even when session store is down.

**bff/src/index.js** — Entry point. Steps in exact order:
1. `require('./config')` first — if env vars are missing the process exits here with a clear error before anything else runs
2. Create `const app = express()`
3. `app.set('trust proxy', 1)` — MUST come before session middleware. Behind Traefik, Express receives HTTP (TLS terminated at Traefik). Without this, `req.secure` is false and the `secure: true` session cookie is never sent to the browser. This will silently break Phase 34 auth.
4. `app.use(express.json())`
5. `app.use('/bff', require('./routes/health'))` — health route mounted at this stage (session middleware added in Plan 03)
6. `app.listen(config.port, () => console.log(`BFF listening on :${config.port}`))` — use config.port, not hardcoded 3001

No error middleware needed for Phase 33. Express 5 auto-forwards rejected async promises to default error handler.
  </action>
