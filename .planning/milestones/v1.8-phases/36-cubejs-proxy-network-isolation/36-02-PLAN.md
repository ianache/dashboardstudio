# Plan: 36-02-PLAN - Network Isolation & Docker Config

Isolate the backend services (FastAPI and CubeJS) from the public network, ensuring they are only reachable via the BFF.

## Tasks

<task id="36-02-01">
  <name>Remove Backend Public Exposure (Traefik)</name>
  <files>["docker-compose.yaml"]</files>
  <action>Remove Traefik labels from the backend service. Ensure only bff and frontend-app are exposed via Traefik.</action>
  <verify>
    <automated>grep -v "traefik" docker-compose.yaml | grep "backend:"</automated>
  </verify>
  <done>Backend Traefik labels removed.</done>
</task>

<task id="36-02-02">
  <name>Remove Backend Public Exposure (Ports)</name>
  <files>["environment/docker-compose.yml"]</files>
  <action>Remove ports mappings for backend (8000), cubejs (4000, 3000), postgres (5432), and redis (6379).</action>
  <verify>
    <automated>grep -v "ports:" environment/docker-compose.yml | grep -E "backend:|cubejs:|postgres:|redis:"</automated>
  </verify>
  <done>Public port mappings removed from environment services.</done>
</task>

<task id="36-02-03">
  <name>Network Audit</name>
  <files>["docker-compose.yaml", "environment/docker-compose.yml"]</files>
  <action>Audit all services to ensure no accidental public exposure. Confirm all internal services share the biportal-network.</action>
  <verify>
    <automated>grep "biportal-network" environment/docker-compose.yml</automated>
  </verify>
  <done>Network audit completed and verified.</done>
</task>
