# Plan: 35-02-PLAN - CORS Consolidation

## Goal
Make the BFF the sole owner of CORS headers and remove CORS handling from the FastAPI backend.

## Proposed Changes

### 1. `bff/package.json`
- Add `cors` dependency.

### 2. `bff/src/index.js`
- Configure `cors` middleware.
- Allow the SPA origin.

### 3. `backend/app/main.py`
- Remove `CORSMiddleware`.
- Remove manual CORS header injection in `global_exception_handler`.

## Step-by-Step Execution

### Step 1: Install `cors` in BFF
```bash
cd bff
npm install cors
```

### Step 2: Configure CORS in BFF
- Allow `http://localhost:3000`, `http://localhost:5173`.
- Enable `credentials: true`.

### Step 3: Clean up FastAPI
- Remove `origins` list.
- Remove `app.add_middleware(CORSMiddleware, ...)`
- Clean up `global_exception_handler`.

## Verification Plan

### Manual Verification
1. Call `/bff/api/v1/health` from the browser (SPA).
2. Verify `Access-Control-Allow-Origin` matches the SPA origin.
3. Verify `Access-Control-Allow-Credentials: true` is present.
4. Verify NO duplicate CORS headers.
